## 2026-02-22 - 사용자 문제 중심 1차 아이데이션

### Context
- 사용자는 일기를 꾸준히 쓰고 싶지만, 머릿속 내용을 문장으로 정리하고 다듬는 데 시간이 많이 들어 습관이 끊김.
- 시간이 거의 없는 날에도 기록을 남길 수 있는 매우 낮은 입력 비용이 필요함.
- 동시에 결과물은 사용자 말투와 취향을 반영한 자연스러운 일기여야 함.

### Ideas
- 입력 방식 2트랙:
- 대화형: 에이전트가 질문하며 친구처럼 오늘의 사건/감정/의미를 끌어냄.
- 초간단형: 키워드만 받아 즉시 일기 초안을 생성함.
- 개인화 장치:
- 사용자 문체 프리셋(예: 담백/감성/유머)과 사용자 few-shot 예시 문장을 프롬프트에 반영함.
- 결과물 워크플로:
- 생성 후 사용자가 빠르게 수정하고 저장할 수 있게 하며, 수정 패턴은 추후 개인화 개선 신호로 활용함.

### Selected Direction
- 첫 우선순위 문제를 `문장 작성 부담으로 인한 일기 습관 중단`으로 확정함.
- MVP의 기본 모드는 `키워드 기반 즉시 생성`으로 선택함.
- 품질 보완을 위해 `대화형 질문 모드`를 선택 기능으로 추가하는 방향으로 정함.
- 핵심 가치 문장은 `짧은 단서만으로도 내 스타일의 일기를 빠르게 완성해준다`로 정리함.

### Next Actions
- MVP 사용자 플로우를 `키워드 입력 → 생성 → 수정 → 저장`으로 구체화함.
- 입력 스키마(키워드, 감정, 사건, 문체 프리셋, few-shot)를 정의함.
- 개인화 프롬프트 정책(우선순위, 금지 표현, 길이/톤 제약)을 작성함.
- 초기 품질 지표(생성 시간, 사용자 수정량, 재사용 의향)를 설정함.

## 2026-02-22 - 일기 내 통합 추세 기록 기능 아이데이션

### Context
- 사용자는 감정 상태, 몸무게 변화, 공부 시간, 운동 시간 같은 정량 지표를 기록하고 싶지만, 별도 입력 흐름은 피로도가 높아 지속성이 낮아짐.
- 사용자마다 기록하고 싶은 지표가 달라 고정 템플릿만으로는 만족시키기 어려움.
- 일기 작성 과정에서 자연스럽게 함께 기록되고 관리되는 통합 경험이 필요함.

### Ideas
- 사용자 정의 지표 슬롯:
- 기본 지표(감정/체중/공부/운동)를 제공하되, 사용자 임의 지표를 추가할 수 있게 함.
- 일기 문장 기반 자동 추출:
- `오늘 2시간 공부`, `몸무게 71.2kg` 같은 문장에서 수치/시간 후보를 인식해 지표 값으로 제안함.
- 저장 전 확인 가능한 보정 UX:
- 자동 추출된 값은 저장 전에 사용자가 확인/수정할 수 있도록 지표 카드에서 즉시 편집 가능하게 함.
- 회고 연결:
- 주간/월간 단위로 지표 추세와 일기 요약을 함께 보여주어 패턴 이해를 돕는 인사이트를 제공함.

### Selected Direction
- 2차 기능의 핵심 문제를 `분리 기록 피로로 인한 추세 기록 중단`으로 확정함.
- 구현 우선순위를 `일기 작성 중 지표 자동 추출 + 저장 시 동시 기록`으로 설정함.
- 초기 범위는 사용자 정의 지표 3~5개 등록, 자동 추출 제안, 수동 보정, 일기와 동시 저장으로 제한함.

### Next Actions
- 지표 데이터 모델(`name`, `type`, `unit`, `daily_value`)을 정의함.
- 숫자/시간/감정 스케일에 대한 자동 추출 규칙과 예외 케이스를 정리함.
- 저장 UX를 `자동 반영`과 `확인 후 반영` 중 하나로 결정함.
- 회고 화면의 최소 범위를 주간 추세 카드 중심으로 정의함.

## 2026-02-22 - 주간/월간/연간 인사이트 리포트 기능 아이데이션

### Context
- 사용자는 기록을 남기는 것에서 끝나지 않고, 주간/월간/연간 단위로 의미 있는 회고를 원함.
- 단순 요약만으로는 재방문 동기와 행동 변화 유도가 약함.
- 기록 데이터와 추세 데이터를 결합해 해석 가능한 인사이트와 다음 행동 제안을 제공해야 함.

### Ideas
- 기간별 리포트 구조:
- 주간은 하이라이트, 감정/루틴 변화, 다음 주 실험 1개 중심으로 구성함.
- 월간은 반복 패턴, 유지된 습관, 에너지 소모 요인을 중심으로 구성함.
- 연간은 전환점, 장기 변화 테마, 성장 스토리 중심으로 구성함.
- 인사이트 생성 방식:
- 관찰(무슨 일이 있었는지), 근거(데이터/기록), 의미(왜 중요한지), 행동(다음 한 가지) 형식을 사용함.
- 신뢰 장치:
- 인사이트마다 근거 범위(예: 최근 4주 중 3주)와 확신도(높음/중간/탐색)를 표시함.
- 과해석을 방지하기 위한 주의 문구를 포함함.

### Selected Direction
- 3차 기능 목표를 `요약을 행동 가능한 자기이해 리포트로 전환`으로 확정함.
- 초기 범위는 주간/월간 인사이트 리포트와 실행 제안까지 포함하고, 연간 리포트는 데이터 축적 후 2단계로 분리함.
- 출력 포맷은 `관찰 3개 + 해석 2개 + 다음 행동 1개`를 기본 템플릿으로 채택함.

### Next Actions
- 인사이트 카드 템플릿(`관찰-근거-의미-다음 행동`)을 정의함.
- 기간별 최소 데이터 조건(주간/월간/연간 생성 기준)을 정함.
- 확신도 라벨 기준과 과해석 방지 정책 문구를 작성함.
- 리포트 UI 최소 범위(카드 수, 요약 문장, 시각화 요소)를 정리함.
