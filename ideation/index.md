# 아이데이션 인덱스

## 프로젝트 포커스
- echodiary 프로젝트 아이데이션

## 현재 요약
- 핵심 문제를 `문장화 부담으로 일기 습관이 끊기는 문제`로 확정함.
- MVP 기본 방향을 `키워드 기반 즉시 일기 생성`으로 정함.
- 보조 방향으로 `대화형 질문 기반 회상/작성`을 추가 검토함.
- 2차 기능 방향으로 `일기 작성 흐름 내 정량 지표(추세) 통합 기록`을 확정함.
- 3차 기능 방향으로 `주간/월간/연간 인사이트 리포트`를 정의함.

## 활성 스레드
- 키워드 입력 스키마와 문체/few-shot 개인화 규칙 설계.
- 대화형 모드 질문 흐름(턴 수, 질문 타입, 종료 조건) 설계.
- 생성 품질 지표와 수정 피드백 반영 방식 정의.
- 사용자 정의 지표 모델 및 자동 추출/보정 UX 설계.
- 기간별 리포트 인사이트 템플릿과 확신도/근거 표시 기준 설계.

## 열린 질문
- MVP에서 대화형 모드를 동시에 포함할지, 2차 릴리스로 분리할지?
- 개인화 강도를 어디까지 허용할지(원문 보존 vs 자연스러운 재작성)?
- 최소 입력 항목은 무엇인지(키워드만 필수, 감정/사건은 선택)?
- 지표 저장 시 사용자 확인 단계를 기본으로 둘지, 자동 반영을 기본으로 둘지?
- 회고 화면에서 첫 릴리스에 포함할 지표 시각화 범위는 어디까지인지?
- 연간 리포트 생성의 최소 데이터 조건을 어떻게 설정할지?
- 인사이트 제안의 개입 강도를 어디까지 허용할지(제안 중심 vs 코칭 중심)?
